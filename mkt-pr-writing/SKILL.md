# PR 기사 빌더 (뉴스와이어 작성요령 + 검토용 목업)

두 가지를 만든다. ① **배포용 원고**(뉴스와이어 폼에 붙여넣을 텍스트, 공식 작성요령 강제) ② **검토용
HTML 목업**(실제 매체에 실린 것처럼 렌더, 상단에 **"INTERNAL REVIEW — 발행 전 검토용"** 배너 고정).
PDF가 필요하면 `make-pdf` 스킬로 변환.

## 시작 전 필독 (작성요령 로드)

배포용 원고를 쓰거나 보도자료 본문을 손볼 때는 **반드시 `references/newswire-style.md`를 먼저 읽고**
그 규칙을 강제한다. 아래는 요약이며, 전체·정확한 규칙과 출처는 그 파일에 있다.

### 뉴스와이어 핵심 규칙 (요약 — 전체는 references/newswire-style.md)

- **제목**: `?`/`!`·연속기호 금지, 말줄임은 `…`. 핵심이 제목에. 다 못 담으면 부제목으로 보완.
- **리드 = 육하원칙 요약** (결론·전망부터 X). 본문 중요도순, 문단당 한 메시지.
- **문장 60자 이내·한 문장 한 메시지·능동태·3인칭**(1인칭 '우리/저희' 금지).
- 회사명 **`(주)`·주식회사 제거**, 첫 등장 후 **약칭**. 인물 첫 풀네임+직책→이후 성+직책.
- 외래어 첫 등장 **`한글(원어)`**(클로드 코드(Claude Code)), 약어 첫 등장 **`한글뜻(약어)`**(건설사업관리(CM)).
- 숫자 9999까지 아라비아, 만 단위 콤마(`1만5000명`), 날짜 `OO월 OO일`(띄어쓰기), 범위 `~`.
- **인용 1개+**(실명·직책), **회사소개(boilerplate)**, **연락처(이름·전화·이메일) 필수**.
- 사진: 보도 적합·고해상도·가로·인물/현장컷(로고 단독 지양). 화면 보안·초상권 확인.
- **금지**: 느낌표·과장어(대박/폭발적/100%/최고/유일/무조건), 단순광고, 지어낸 사실·매체사칭.

## PR 5대 요소 (작성 전 체크)

글을 쓰기 전에 다음 5가지를 확인하고, 답이 없으면 사용자에게 질문한다.

| 요소 | 확인 질문 | 글에 어떻게 반영하나 |
|------|-----------|----------------------|
| **① 목표 설정** | 이 보도자료로 무엇을 달성하려 하는가? (인지도·신뢰도·행동 유도 중 무엇이 우선?) | 목표가 제목·리드·CTA에 일관되게 반영됐는지 점검 |
| **② 명확한 제목** | 독자가 제목만 봐도 핵심을 이해할 수 있는가? | 핵심 정보 우선 배치, 다 못 담으면 부제목으로 보완 |
| **③ 핵심 메시지** | 독자에게 딱 하나만 기억시킨다면 무엇인가? | 리드·인용·소제목에 반복 강조, CTA 명시 |
| **④ 사실 기반** | 수치·사실·인용 출처가 모두 검증됐는가? | 과장어·미확인 수치 제거, 실명+직책 인용 |
| **⑤ 타깃 대상층** | 누가 읽는 보도자료인가? (일반 소비자·B2B·투자자·개발자 등) | 배포 매체·톤·용어 난이도 조정 |

→ 5가지 중 하나라도 불명확하면 **작성 전에 사용자에게 먼저 확인**한다.

## 워크플로

1. **PR 5대 요소 확인** — 목표·제목·메시지·사실·타깃이 모두 명확한지 점검. 모르면 질문.
2. **작성요령 로드** — `references/newswire-style.md`를 읽는다(배포 원고/본문 작성 시 필수).
3. **배포용 원고 작성/수정** — `*-newswire-submission.md`로 저장. §14 체크리스트 자가 점검 후 제시.
4. (요청 시) **검토용 HTML 목업** — 아래 골격으로 렌더. 본문은 3의 원고를 쓰되 목업 장식은 원고에 역유입 금지.
5. (요청 시) **PDF** — `make-pdf` 또는 HTML을 headless Chrome으로 인쇄.
6. **사진** — 보도 적합 고해상도 가로컷 우선(§11). 로고는 보조 1장.

## 산출 골격 (순서 고정 — 검토용 HTML 목업)

1. **검토 배너** — 빨강 고정 바, "INTERNAL REVIEW · 발행 전 검토용"
2. **매체 헤더** — `{매체명}` 로고 + 메뉴 (모바일 뉴스 레이아웃)
3. **카테고리 네비 / breadcrumb** — `[카테고리] > [서브카테고리]`
4. **기사 제목** — 한 줄, `word-break: keep-all`
5. **발행/업데이트 일시** — `YYYY-MM-DD HH:MM` 2개
6. **대표 이미지 + 캡션** — `[사진= {제공주체} 제공]`
7. **본문** — §아래 기사체 규칙
8. **기자 라인** — `{기자명} 기자  {이메일}`
9. **관련기사 / (댓글) / 매체 푸터** — `{매체 사업자정보}` placeholder

## 본문 작문 규칙 (기사체 — 광고체 아님)

- **리드 문단 = 육하원칙 요약**: 누가·무엇을·언제·왜 했는지 한 문단에.
- **둘째 문단 = 배경/맥락**, 이후 `h3` 소제목으로 단락 구획.
- **인용 1개 이상**: `{이름} {직책}은 "…"고 말했다` 형태로 본문 `<p>` 안에 녹인다. blockquote 풀쿼트 형식 금지. 실명·직책은 사실만.
- **마무리 = 향후 전망/기대효과.**
- **톤: 객관 보도체.** "100% / 최고 / 유일 / 무조건" 등 광고·과장 표현 금지(기사 신뢰·표시광고법).

## 디자인 토큰 (매체 느낌)

- 모바일 우선(`max-width: 100vw`), `Noto Sans KR`, 본문 17px·line-height 1.8, `keep-all`.
- 액센트 컬러 1개(링크·인용 보더). 검토 배너만 빨강.
- 선택: 스크롤 시 상단 고정헤더 + 읽기 진행바(완주 체감용) — 검토 편의 기능.

## 본문 허용 요소 (이것만 사용)

실제 기사에 삽입 불가한 커스텀 UI는 HTML 목업에도 넣지 않는다.

**허용**: 텍스트(`p`·`h3`) + 이미지 플레이스홀더(`figure`)
**금지**: 통계 카드, 표(`table`), 인포그래픽, 뱃지, 강조 박스, 불릿 나열(`ul`·`li`) 등 커스텀 컴포넌트

수치·항목 나열은 본문 `p` 안에 자연스럽게 녹여 쓴다.
(예: "교육은 A부터 B, C, D 순으로 진행됐다." / "참가자 19명 전원이 5점 만점에 4.74점을 매겼다.")

## placeholder 규칙 (반드시 치환)

- `{매체명}`, `{제공주체}`(사진), `{기자명}`/`{이메일}`, `{매체 사업자정보}`(주소·등록번호·발행인).
- 본문/제목/인용/소제목은 `[…]` 자리표시로 비워두고 원고로 채운다.
- 매체·인물·수치는 **사실 확인된 것만**. 허위 기자명·미게재 매체 사칭 금지.

## 산업분야·주제 추천

배포용 원고 완성 후 뉴스와이어 산업분야와 주제 선택값을 제안한다.

**산업분야** — 최대 2개 선택. 조회율(트래픽)과 타깃 적합도를 함께 고려해 우선순위 높은 2개를 고른다. IT > 인공지능처럼 트래픽이 높은 카테고리를 1순위로 잡고, 기사의 실제 독자층이 모이는 카테고리를 2순위로 선택.
주요 카테고리: IT(인공지능·소프트웨어·보안 등) / 경제(기업 경영·창업·인재와 고용 등) / 교육(교육 일반·직업교육·온라인 교육 등) / 정부(공공기관·중앙정부·비영리조직 등) / 문화·생활·건강·금융·운송·환경 등

**주제** — 라디오 버튼(1개 선택). 기사의 핵심 뉴스 가치 기준으로 고른다.
주요 옵션: 실적 / 신상품 / 사업 확장 / 인사 / 수상 / 조사분석 / 행사 / 회사 공지 / 설립 / 제휴 등

형식:
```
[산업분야·주제 추천]
산업분야: {분야1 > 세부}, {분야2 > 세부}, {분야3 > 세부}
주제: {주제} — 이유: {한 줄}
```

## 키워드 추천

배포용 원고 완성 후 키워드를 제안한다. 뉴스와이어 배포 시 검색 노출(구글·네이버·빙)에 활용.

**추천 기준 (우선순위 순)**
1. **핵심 키워드** — 기사 주제를 직접 표현하는 검색어 (예: AI 교육, 공공기관 AI, 회사명)
2. **확장 키워드** — 관련 업계·트렌드 용어로 노출 범위 확대 (예: AX 교육, 디지털 전환, 맞춤형 교육)

형식:
```
[키워드 추천]
핵심: {키워드1}, {키워드2}, {키워드3}
확장: {키워드4}, {키워드5}, {키워드6}
```

## make-pdf 연동

HTML 확정 → `make-pdf` 스킬로 검토용 PDF 출력(공유·결재용). 이 스킬은 **HTML 목업**까지 책임.

## blog-thumbnail-3color / make-pdf 와 구분

- **이 스킬** = 기사 *전문* 목업(HTML, 기사체 본문 포함).
- `blog-thumbnail-3color` = 썸네일 *이미지*. `make-pdf` = *렌더/변환* 단계. 역할이 다르다.

## 합격 기준

**PR 5대 요소 점검** (배포 원고 제시 전 자가 체크):
- [ ] ① 목표: 이 원고가 달성하려는 목표가 제목·리드·CTA에 일관되게 반영됐는가
- [ ] ② 제목: 제목만으로 핵심이 전달되는가
- [ ] ③ 핵심 메시지: 독자가 하나만 기억할 메시지가 리드·인용·소제목에 명확히 드러나는가
- [ ] ④ 사실 기반: 모든 수치·사실·인용에 출처가 있고 과장어가 없는가
- [ ] ⑤ 타깃: 배포 매체와 톤이 목표 독자층에 맞게 조정됐는가

**배포용 원고**: `references/newswire-style.md` §14 체크리스트를 그대로 점검(제목 핵심 포함·문장 60자·(주) 제거·
외래어 한글(원어)·숫자/날짜·인용·회사소개·**연락처 입력**·과장어 없음·사진 규격·이중 카테고리).

**검토용 HTML 목업**:
- [ ] 검토 배너가 상단에 고정됐는가(실제 기사 오인 방지)
- [ ] 본문이 위 뉴스와이어 규칙(리드 육하원칙·인용·객관톤)을 따르는가
- [ ] 목업 장식(가짜 광고·헤더/댓글)이 *배포 원고*에는 들어가지 않았는가
- [ ] placeholder(`{매체명}`·`{기자명}`·`{매체 사업자정보}` 등) 미치환분이 없는가
- [ ] 일시가 `YYYY-MM-DD HH:MM` 형식인가

## 자주 하는 실수

- 보도톤이 아니라 광고 카피처럼 씀 (X — 객관체, 과장어 금지)
- 검토 배너 없이 실제 기사처럼 보이게 함 (X — 발행 전 오인·사칭 위험)
- 리드에 결론·전망부터 씀 (X — 리드=육하원칙 요약, 전망은 마무리)
- 매체·기자·수치를 지어냄 (X — 사실만, 미게재 매체 사칭 금지)
- PDF 변환 로직을 이 스킬에 욱여넣음 (X — `make-pdf` 담당)
