# 뉴스와이어 공식 작성요령 (배포용 보도자료 강제 규칙)

> 출처(모두 newswire.co.kr 공식 가이드):
> - 보도자료 작성법 `?ed=4` · 보도자료 만들기 `?ed=244`
> - 편집 가이드라인 `?ed=8` · **뉴스 스타일 가이드 `?ed=361`**(v4, 2024-05-22)
> - 어떻게 배포하나 `?sd=48` · 배포 요금 `?sd=11` · FAQ `?ed=3&cat=13`
>
> 이 파일의 규칙은 **뉴스와이어로 배포할 원고**를 쓸 때 강제한다. 검토용 HTML 목업도 본문은 이 규칙을 따른다.

---

## 0. 산출물 2종 (혼동 금지)

| 산출물 | 무엇 | 규칙 |
|---|---|---|
| **배포용 원고** (`*-newswire-submission.md`) | 뉴스와이어 폼에 붙여넣을 텍스트 | 이 파일 전부 강제 |
| **검토용 HTML 목업** (`*-article*.html`) | 매체에 실린 것처럼 보이는 시안 | 본문은 이 규칙 + 상단 검토배너 + 목업 장식(가짜 광고·헤더)은 원고에 넣지 않음 |

목업의 네이티브 광고·중간 광고·매체 헤더/푸터/댓글은 **디자인 장식**이다. 배포 원고에는 절대 포함하지 않는다.

## 1. 보도자료 구성요소 (순서)

제목 → 부제목 → (발행)날짜 → 출처(회사명) → 리드 → 본문 → 인용 → 회사소개(boilerplate) → 연락처 → 사진/동영상.

## 2. 제목

- 핵심이 제목에 바로 드러나야 한다.
- **물음표·느낌표 금지**, 연속 기호 금지. 말줄임은 마침표 3개가 아니라 `…`.
- 제목으로 다 못 담으면 **부제목**으로 보완.

## 3. 리드(첫 문단)

- **육하원칙 요약**: 누가·무엇을·언제·어디서·왜·어떻게. 결론/전망부터 쓰지 않는다.
- 이후 본문은 **중요도순** 배열, **문단당 하나의 메시지**.

## 4. 문장·문체

- **한 문장 = 한 메시지. 문장 60자 이내.**
- **능동태** 우선, 피동("~된다") 회피.
- **3인칭 관찰자 시점.** '우리는/저희/나는' 등 1인칭 금지.
- "말하듯이" 자연스럽게, 명확·직설.

## 4-1. 동어 반복 줄이기 (문장 간결화)

> 출처: 뉴스와이어 블로그 '기사 문장을 간결하게 만드는 동어 반복 줄이기'(newswire.co.kr `?p=18963`).

같은 단어·서술어·조사가 겹치면 문장이 늘어진다. 아래 5가지로 다듬는다.

- **① 반복 서술어 하나로 묶기.** "기술력이 뛰어난 곳으로, 성장 가능성이 높은 곳이다" → "기술력이 뛰어나고 성장 가능성도 높다"
- **② 같은 단어 중복 쓰임 줄이기.** "누가 조사해도 같은 조사 결과가 나온다" → "누가 조사해도 같은 결과가 나온다"
- **③ 조사·연결 표현 반복 제거.** "반응에 대해 …에 대해 논의했다" → "반응을 두고 논의했다"
- **④ 반복 서술어 표현 바꾸기.** "관계를 맺기 위해 협약을 맺었다" → "관계를 형성하기 위해 협약을 체결했다"
- **⑤ 반복 명사 다른 표현으로 전환.** "삶을 실천한 그의 삶" → "삶을 실천한 그의 면모"

**단, 핵심 용어·고유명사는 정확성을 위해 반복 유지.** 억지로 바꿔 문장이 부자연스러워지지 않게 한다.

## 5. 회사명·인명·직책

- 회사명에서 **'주식회사'·'(주)' 생략.** (예: `(주)선엔지니어링종합건축사사무소` → `선엔지니어링종합건축사사무소`)
- 본문 첫 등장은 정식명, 이후 **약칭**. (예: `…종합건축사사무소(이하 선엔지니어링)` → 이후 `선엔지니어링`)
- 인물: 첫 등장 **풀네임+직책**, 재등장은 **성+직책**(예: 오정근 상무 → 오 상무).
- 일반인 이름 뒤 `씨`, 미성년자는 `군/양`.

## 6. 숫자·날짜·시간·단위

- **9999까지 아라비아 숫자.** 만 이상 한글 단위 + 콤마: `13억3890만원`, `1만5000명`(❌ `1만 5천 명`).
- 나이: 숫자+`세`(5세), 한글+`살`(두 살).
- 날짜/시간: `OO월 OO일 OO시 OO분`(띄어쓰기). 범위는 `~`(앞뒤 붙임). 해외 일정 `2일(현지시간)`.

## 7. 외래어·약어

- 외래어 **첫 등장 = 한글음차 + (원어)**: `클로드 코드(Claude Code)`, `앤트로픽(Anthropic)`, `챗GPT 엔터프라이즈(ChatGPT Enterprise)`. 이후엔 한글만.
- 이미 정착된 표기는 한글만: 구글, 바이든, 캘리포니아.
- 약어: **첫 등장 한글 뜻 + (약어)**. (예: 건설사업관리(CM), AI 전환(AX))

## 8. 인용문

- **1개 이상.** 형식 `"…" — 이름 직책` (실명·직책은 사실만).
- 주관적 주장은 반드시 인용부호 + 코멘트로 처리(본문 단정 금지).

## 9. 회사소개(boilerplate)

- 본문 끝에 **연혁·사업·상품·규모**를 요약해 넣는다(권장, 사실상 필수).

## 10. 연락처 (필수)

- 보도자료 **맨 끝에 문의용 이름·전화번호·이메일**을 반드시 넣는다. 없으면 반려/문의 불가.

## 11. 사진·멀티미디어

- 사진·동영상·인포그래픽 첨부 시 **언론 주목·조회수↑**.
- **언론 보도에 적합한 품질**이어야 하며 저해상도·편집 미달은 **삭제** 대상.
- 실무: 가로(landscape)·고해상도(가로 1200px+, 클수록 좋음)·인물/현장컷 우선. 로고·슬라이드 캡처만으론 픽업 약함.
- 화면에 사내 기밀·개인정보가 찍힌 컷은 보안 확인(필요 시 제외/블러). 초상권 동의 확인.

## 12. 금지 표현 (편집 반려 사유)

- 느낌표·과장 형용사. `대박·폭발적인·수익 보장·100%·최고·유일·무조건` 등 광고 문구.
- 뉴스 가치 없는 **단순 상품 광고**.
- 허위 기자명·미게재 매체 사칭, 지어낸 수치/인물.

## 13. 배포 운영 (요금·타겟·타이밍)

- 등급(부가세 포함, 1건): 베이직 66,000 / **스탠다드 176,000** / 프리미엄 297,000.
  - 사진 1/3/5장. 유튜브 영상은 전 등급 제공.
  - **담당분야 기자 이메일 직접 발송 + 보도 모니터링은 스탠다드부터.** 베이직은 제휴매체 자동노출만.
- 타겟: 345개 산업분야 / 371개 지역. 제휴 4,300개 편집국, 43,000 언론인. **이중 카테고리**로 좁혀 발송(예: 건설/엔지니어링 + IT·AI).
- 노출: 구글·네이버·빙·줌 검색. 포털 뉴스 노출은 **제휴 기자 픽업**으로 발생 → 직접발송(스탠다드+)이 실효.
- 타이밍: **평일 오전 8시~오후 2시** 배포. 오후 3시 이후·금요일 오후 회피. **전날 예약**으로 편집자 사전 검수.
- 신선도: 발표일과 배포일 간극이 크면 올드 뉴스. 배포 시점 기준 후크로 리드를 맞춘다.

## 14. 배포 원고 합격 체크리스트 (제출 전 자가 점검)

- [ ] 제목에 핵심 포함, `?`/`!` 없음
- [ ] 리드가 육하원칙 요약 (결론·전망 아님)
- [ ] 모든 문장 60자 이내·한 문장 한 메시지·능동태·3인칭(1인칭 없음)
- [ ] 동어 반복(같은 서술어·단어·조사) 없이 문장이 간결한가
- [ ] 회사명 `(주)` 제거 + 첫 등장 후 약칭
- [ ] 외래어 첫 등장 `한글(원어)`, 약어 첫 등장 `한글뜻(약어)`
- [ ] 숫자/날짜/단위 표기 규칙 준수(만 단위 콤마, 날짜 띄어쓰기, `~`범위)
- [ ] 인용 1개+ 실명·직책, 주관 주장은 인용 처리
- [ ] 회사소개(boilerplate) 포함
- [ ] **연락처(이름·전화·이메일) 입력** (플레이스홀더 잔존 금지)
- [ ] 과장·광고 문구·지어낸 사실 없음
- [ ] 사진 고해상도·가로·보도적합(로고 단독 지양), 화면 보안·초상권 확인
- [ ] 카테고리 이중 타겟, 평일 오전 배포·전날 예약 계획
