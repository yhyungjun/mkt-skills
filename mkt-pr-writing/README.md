# mkt-pr-writing

보도자료/PR 기사를 **뉴스와이어 공식 작성요령에 맞춰 작성**하고, 필요 시 **실제 한국 IT 매체
기사처럼 렌더한 "발행 전 검토용" 목업(HTML)**으로 보여주는 스킬. 명령: `/mkt-pr-writing`.
(구 `pr-article-mockup` 을 레포로 이관·개명한 것)

## 무엇을 하나
두 가지를 만든다.
- **배포용 원고** — 뉴스와이어 폼에 붙여넣을 텍스트. 공식 작성요령을 강제한다
  (제목 핵심 우선·리드 육하원칙·문장 60자·`(주)` 제거·외래어 `한글(원어)`·숫자/날짜 표기·
  인용·회사소개·연락처·사진 규격·과장어 금지). `*-newswire-submission.md`로 저장.
- **검토용 HTML 목업** — 실제 매체 기사처럼 렌더하되 상단에 "INTERNAL REVIEW · 발행 전 검토용"
  빨강 배너를 고정해 발행·사칭 오인을 막는다.

작성 전 **PR 5대 요소**(목표·제목·핵심 메시지·사실 기반·타깃)를 점검하고, 불명확하면 먼저 질문한다.
완성 후 뉴스와이어 **산업분야·주제·키워드**를 추천한다.

## 사용
"이 내용으로 보도자료 써줘" / "뉴스와이어 배포 원고" / "IT 매체 기사 스타일로 목업 보여줘" 처럼
소재만 주면 작성요령을 강제한 원고(→ 필요 시 HTML 목업)를 만든다.
PDF가 필요하면 `make-pdf` 스킬로 변환한다.

전체 규칙·산출 골격은 `SKILL.md`, 뉴스와이어 상세 작성요령은 `references/newswire-style.md` 참고.
블로그 썸네일은 `blog-thumbnail-3color`, 렌더/변환은 `make-pdf` 로 역할이 분리돼 있다.
