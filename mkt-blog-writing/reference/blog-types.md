# Blog Types — 유형별 입력 스펙 + BlogType enum 매핑

---

## BlogType 매핑표

| 사용자 표시 이름 | 코드 enum (BlogType) | 설명 | 키워드 단계 |
|---|---|---|---|
| 정보제공용 | `INFO` | 유용한 정보 전달, 전문성 구축 | `info_search`, `solution_search` |
| 자사홍보용 | `PURCHASE` | 자사 서비스/제품 홍보, 문의 유도 | `provider_search`, `review_compare` |
| 포트폴리오 | `PORTFOLIO` | 고객 성공사례 소개, 신뢰 구축 | `provider_search`, `review_compare` |
| 일상글 | `DAILY` | 자연스러운 브랜딩 이야기 | 키워드 없음 (Step 2 생략) |

## 공통 수집 항목

모든 블로그 유형에서 공통으로 수집한다.

| 필드명 | 설명 | 필수 여부 |
|---|---|---|
| `company_name` | 회사명 | 필수 |
| `company_service` | 회사가 하는 일 (1~2문장) | 필수 |
| `strengths` | 강점 목록 (제목 + 한 줄 설명, 1~3개) | 선택 |

**강점 수집 방식**: 각 강점을 제목(한 줄)과 설명(한 줄)으로 입력받는다.
```
예:
- 제목: "10년 경력 개발팀"
- 설명: "정보올림피아드 수상자 출신 개발자들이 직접 코드를 작성합니다"
```

> **참고**: 강점은 회사 등록 시 `image_urls(최대 4장)`, `image_layout`, `is_active` 메타데이터를 갖는다.
> 원본: `back/src/api/entity/company/company_strength_entity.py:35`

## INFO — 정보제공용

유용한 정보를 전달하여 독자의 문제 해결을 돕고, 전문성과 신뢰를 쌓는다.

### 수집 질문

1. "어떤 주제에 대한 정보를 전달할 예정인가요?"
2. "독자에게 전달할 핵심 정보/내용을 자유롭게 써주세요. (최대 3,000자)"
   - 전문 지식, 노하우, 데이터, 사례 등 모두 포함 가능

### 입력 스키마

```
{
  "blog_type": "INFO",
  "company_name": str,
  "company_service": str,
  "strengths": [{"title": str, "description": str}] | None,
  "user_provided_info": str  # 전달할 핵심 정보 (최대 3,000자)
}
```

### 글 목적

정보 제공 → 독자의 문제 인식 → 전문성/신뢰 구축 → 장기적 브랜딩

## PURCHASE — 자사홍보용

고객이 겪는 문제와 불안을 짚고, 해결책으로 자사를 제시하여 문의/상담/구매를 유도한다.

### 수집 질문

1. "이 블로그를 읽을 타겟 독자는 누구인가요?"
   - 예: "외주 개발을 맡기려는 스타트업 대표"
2. "타겟이 현재 겪고 있는 문제나 고민은 무엇인가요?"
   - 예: "개발사를 선택하기 어렵고, 실패 리스크가 두렵다"
3. "타겟이 우리 서비스/제품을 실제로 이용·체험한 뒤 어떻게 변화하나요?"
   - ⚠️ **이 질문은 글을 읽은 직후의 마음 상태를 묻는 게 아니다**. 사용자가 내 상품을 실제로 써본 뒤 일어나는 **실질적 변화**(역량/결과/지위)를 적는다.
   - ❌ 잘못된 답변 예시 (글을 읽은 직후 결심):
     - "문의해야겠다는 생각이 든다"
     - "이 회사가 믿을 만하다고 느낀다"
     - "상담을 신청하고 싶어진다"
   - ✅ 올바른 답변 예시 (서비스 이용 후 변화):
     - 외주: "막연한 아이디어가 실제 서비스로 런칭되어 매출이 발생한다"
     - 교육: "바이브코딩 과정 수료 후 자기 아이디어를 직접 프로그램으로 만들 수 있는 사람이 된다"
     - 창업: "창업 과정 수료 후 첫 외주 1,000만원 수주로 스스로 돈을 버는 사람이 된다"
   - 사용자가 잘못된 답변(글 읽은 직후 결심)을 하면 다시 물어 서비스 이용 후 변화로 교정한다.

### 입력 스키마

```
{
  "blog_type": "PURCHASE",
  "company_name": str,
  "company_service": str,
  "strengths": [{"title": str, "description": str}] | None,
  "target_audience": str,          # 타겟 독자
  "target_problem": str,           # 타겟이 겪는 문제
  "target_desired_outcome": str    # 원하는 변화/결과
}
```

### 글 목적

타겟 문제 공감 → 리스크 경고 → 자사 강점 제시 → 결정 유도 (비교 피로 해소)

## PORTFOLIO — 포트폴리오

실제 고객 성공사례를 통해 자사 실력을 간접 증명하고, 구매를 망설이는 잠재 고객의 신뢰를 얻는다.

### 수집 질문

1. "소개할 고객이 처음에 겪던 문제는 무엇인가요?"
   - 예: "기존 ERP 시스템이 너무 느리고 오류가 잦아 업무 효율이 떨어졌다"
2. "어떻게 해결했나요? (자사의 솔루션/서비스)"
   - 예: "맞춤형 클라우드 ERP를 3개월 만에 구축했다"
3. "고객이 얻은 결과/변화는 무엇인가요?"
   - 예: "업무 처리 속도 40% 향상, 오류율 0%"

### 입력 스키마

```
{
  "blog_type": "PORTFOLIO",
  "company_name": str,
  "company_service": str,
  "strengths": [{"title": str, "description": str}] | None,
  "customer_problem": str,   # 고객이 겪던 문제
  "solution": str,           # 자사의 해결 방법
  "result": str              # 고객의 변화/결과
}
```

### 글 목적

고객 문제 공감 → 해결 과정 스토리텔링 → 결과 증명 → 잠재 고객 행동 유도

## DAILY — 일상글

자연스러운 브랜드 이야기를 통해 독자와 감성적 연결을 만든다.

### 수집 질문

1. "어떤 소주제로 쓸까요? 아래 중 선택해주세요:"
   - `customer_gift`: 고객이 주신 선물 (후기, 감사 표현 등)
   - `passion`: 일에 대한 열정 (창업 이야기, 노력 과정)
   - `vision`: 앞으로의 비전 (목표, 꿈, 방향성)
   - `routine`: 일상/루틴 (하루 일과, 업무 방식, 팀 문화)
2. "이야기를 자유롭게 써주세요. (최대 2,000자)"
   - 실제 있었던 일, 느낀 점, 생각 등 자유롭게 기술

### 소주제 목록 상세

| 소주제 코드 | 설명 | 예시 |
|---|---|---|
| `customer_gift` | 고객이 주신 선물 | 감사 메시지, 재구매 스토리, 후기 공유 |
| `passion` | 일에 대한 열정 | 창업 계기, 어려움 극복, 성장 스토리 |
| `vision` | 앞으로의 비전 | 올해 목표, 서비스 방향, 사회적 가치 |
| `routine` | 일상/루틴 | 팀 문화, 업무 방식, 하루 일과 공유 |

### 입력 스키마

```
{
  "blog_type": "DAILY",
  "company_name": str,
  "company_service": str,
  "strengths": [{"title": str, "description": str}] | None,
  "daily_topic": "customer_gift" | "passion" | "vision" | "routine",
  "daily_story": str  # 이야기 (최대 2,000자)
}
```

### 글 목적

브랜드 인간화 → 공감대 형성 → 자연스러운 팔로워 유지

**참고**: DAILY 유형은 Limbic 분석과 키워드 단계(Step 2)를 생략한다.
