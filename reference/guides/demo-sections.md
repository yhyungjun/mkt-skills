# 제품 UI 데모 섹션 (정적 목업으로 "실제 쓰는 모습" 보여주기)

> **출처**: axacademy-landing `Solution` 섹션 — `components/demo/`의 3탭 데모
> (Claude Desktop UI · MCP 커넥터 · company-bot). 강의/제품이 **무엇을 하게 해주는지**를
> 말이 아니라 **화면**으로 보여주는 설득 장치(playbook의 mac-open-motion iframe 데모와 형제).
> **왜**: "할 수 있다"는 카피 100줄보다, 결과물 화면 1개가 예비구매자의 "내가 이걸 만들 수 있겠다"를 만든다.

---

## 패턴 요지
- **정적 목업, 라이브 연결 아님**: 실제 API/제품을 임베드하지 않고, 그 UI를 **재현한 정적 컴포넌트**.
  외부 의존·로딩 실패·인증 없이 항상 똑같이 보인다. (무거운 실데모가 필요하면 playbook의 iframe+맥북 프레임.)
- **탭/시나리오 전환**: 한 섹션에서 여러 사용 장면을 탭으로 전환(예: Claude Desktop / MCP 커넥터 / 사내봇).
  데이터는 `*.data.ts`로 분리(컴포넌트=레이아웃, 데이터=대본). 긴 대화 스크립트를 코드에서 빼낸다.
- **컨테이너 + 자식**: `XxxDemo.tsx`(탭 셸·프레임) + 장면별 자식 컴포넌트. 채팅형이면 메시지 배열을 맵으로 렌더.

```
components/demo/
  ClaudeDesktopDemo.tsx     # 탭 셸(제품 UI 프레임)
  CompanyBotDemo.tsx        # 한 장면(사내봇 대화)
  ClaudeCodeSession.tsx     # 한 장면(터미널/세션)
  claude-sessions.data.ts   # 대본(메시지·스텝) — 컴포넌트와 분리
```

## ⚠️ 개인정보 — 목업엔 실데이터 금지 (필수)
- 목업 대화/화면에 **실명·실제 회사명·연락처·내부 데이터 금지**. 이름은 `사용자`/`User`/익명으로 치환.
- 스크린샷·캡처를 그대로 박지 말고 **재현**한다(우연한 PII 노출 방지). 실제 제품 캡처는 마스킹.
- 브랜드 로고/상표는 사용 권한 확인된 것만.

## 디자인 정합
- 데모 프레임은 다크 배경 섹션(예: `cone-section` 부채꼴 글로우)에 얹으면 "화면이 빛난다" → 시선 집중.
- 본문 디자인 토큰(`--signal` 등) 재사용. 모노 폰트는 코드/터미널 장면에만.
- `prefers-reduced-motion` 존중(자동 타이핑·전환은 모션 줄임 시 정지/즉시표시).

## 언제 쓰나 / 안 쓰나
- **쓴다**: 결과물·도구 사용 장면이 곧 핵심 가치인 강의(코딩·AI·노코드 등). Solution/Curriculum 섹션 보강.
- **안 쓴다**: 오프라인 행사·일반 세미나 등 "화면 결과물"이 핵심이 아닐 때 → 후기·로고·강사로 대체.
