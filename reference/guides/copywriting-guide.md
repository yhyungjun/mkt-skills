# 카피라이팅 가이드 — 강의 사전예약·결제 랜딩

> 강의(온라인/오프라인) 랜딩의 **섹션별 설득 구조·톤**. granter-landing(셀러OS) 실제 카피에서
> 패턴만 추출하고 브랜드 고유값은 `{플레이스홀더}`로 비웠다. 자기 상품으로 **예시만 교체**하면 된다.
> 구조(순서·역할)는 그대로 쓰고, 문구는 갈아끼우는 방식.

---

## 0. 핵심 원칙 (모든 섹션 공통)

1. **한 섹션 = 한 메시지.** 한 화면에서 독자가 가져갈 한 줄이 명확해야 한다.
2. **독자 언어 > 우리 언어.** 기능명("대시보드")보다 독자의 통증("이번 달 얼마 남았는지 모르겠다")으로 말한다.
3. **구체 > 추상.** "효율 향상" ✕ → "엑셀 정리 시간 0" ○. 숫자·기간·결과물 형태로 못 박는다.
4. **결과물 = 자산.** 강의는 "수강"이 아니라 "졸업 후 손에 남는 것"으로 판다. (granter: "수료증이 아니라 매일 쓰는 운영 자산")
5. **CTA는 한 동선.** 페이지 전체에서 1차 CTA는 한 종류(사전예약 or 결제). 여러 버튼이 서로 경쟁하지 않게.

## 1. 섹션별 설득 구조

각 섹션의 **역할 → 카피 공식 → granter 추출 예시 → 교체 템플릿** 순서.

### Hero — 가치제안 한 줄
- **역할**: 3초 안에 "이게 나한테 뭔지" 전달 + 메인 CTA.
- **공식**: `[독자가 원하는 결과]` + `[그걸 얻는 방식 한 줄]` + `[대상·조건 자격]` + CTA.
- granter 예시:
  - H1: `매출·재고·자금 / 한 화면에서 관리하세요`
  - 서브: `스마트스토어·쿠팡 이커머스 셀러를 위한 4주 교육 / 오프라인·온라인 선택 가능 · 비개발자 가능`
- 템플릿:
  ```
  H1:  {핵심 가치 동사형 한 줄}
  서브: {타깃 페르소나}를 위한 {기간} {형식} / {선택지} · {진입장벽 해소}
  CTA: {사전예약하기 | 신청하기} →
  ```
- 팁: H1은 명사 나열이 아니라 **동사로 끝내 행동을 암시**("관리하세요"). 서브줄에 자격 조건("비개발자 가능")을 박으면 이탈 줄고 전환 질 올라감.

### Benefits (사전예약 혜택) — 지금 행동할 이유 3개
- **역할**: 결제 전 단계(사전예약)의 즉시 보상. "지금 등록하면 얻는 것".
- **공식**: 혜택 3개를 `번호 + 제목(명사구) + 한 줄 설명`. 1개는 권위(강사/실물), 1개는 무료 자산, 1개는 우선권/긴급성.
- granter 예시: `01 미리보기 강의 제공` / `02 무료 AI 스킬 제공` / `03 정규 오픈 우선 안내 — 얼리버드 할인 기회를 놓치지 마세요`
- 템플릿: `01 {맛보기 콘텐츠}` · `02 {무료 자산/툴}` · `03 {우선 안내·할인 우선권}`

### CreatorIntro / 브랜드 신뢰도 — 누가 가르치나
- **역할**: 권위 이전(authority transfer). 강사/브랜드의 실적을 짧게.
- **공식**: `[정체성 한 줄] + [검증 가능한 수치/이력 2~3개]`. 과장 금지, 검증 가능한 것만.
- 팁: 수치는 출처 암시("패스트캠퍼스 AI분야 1위", "국내 최대 OO 채널")가 있으면 신뢰↑. 없는 수치 지어내지 말 것.

### WhyNow / ProblemAware — 문제 인식 (통증 언어화)
- **역할**: 독자가 "맞아, 내 얘기다"라고 느끼게. 해결책 전에 **고통을 먼저 또렷이**.
- **공식**: 섹션 헤딩 = 공감 한 줄. 그 아래 통증 3카드 = `[증상 제목] + [내 입에서 나올 법한 혼잣말]`.
- granter 예시:
  - 헤딩: `매출은 늘고있는데, 늘 불안합니다`
  - 카드: `돈이 안 보인다 — 팔수록 바쁜데, 정작 수익은 잘 모르겠어요` / `결정이 늦어진다 — 엑셀 정리하다 시간 다 가고, 판단은 자꾸 미뤄져요` / `현금이 마른다 — 재고에 묶인 돈, 자금이 갑자기 막힐까 늘 불안해요`
- 템플릿: 카드 desc는 **1인칭 구어체**("~어요/~죠")로 — 광고 문구가 아니라 독자 속마음처럼.
- ProblemAware(시각화 버전): 헤딩으로 "흩어짐"을 직격(`매출·정산·자금 화면 / 각각 흩어져 관리하고 있으시죠?`) + 흩어진 화면 비주얼.

### WhoIsThisFor — 페르소나 호명
- **역할**: "이건 너를 위한 거다"라고 **이름표를 붙여** 자기 식별을 유도.
- **공식**: 페르소나 3명 = `[별명형 라벨] + [그 사람의 결정적 통증 한 줄]`.
- granter 예시: `월 300만 이상 셀러 — 매출은 느는데, 이번 달 얼마 남았는지 답하기 어려운 분` / `엑셀에 묶인 운영자 — …정리하느라 시간을 뺏기는 분` / `툴을 못 살리는 셀러 — 여러 도구는 쓰지만 따로따로라…`
- 팁: 라벨은 **직업이 아니라 상태**("엑셀에 묶인 운영자"). desc는 "~한 분"으로 끝내 호명감 강화.

### Solution — 해결책 (한 방에 보여주기)
- **역할**: 문제 직후 "그래서 이렇게 해결된다"를 **데모/비주얼로** 즉시 증명. 말보다 보여주기.
- **공식**: 헤딩 = before→after 전환 한 줄 + 실제 결과물(데모 iframe/스크린샷).
- 팁: 텍스트로 설명하려 들지 말 것. 라이브 데모/목업 하나가 문단 3개를 이긴다.

### Curriculum — 무엇을 배우나 (주차별)
- **역할**: 4주(또는 N주) 동안의 여정을 단계로. 매주 "끝나면 손에 뭐가 남는지".
- **공식**: 주차별 `[주차] + [그 주의 결과물(산출물 중심)]`. "배운다"가 아니라 "만든다/완성한다".
- 팁: 데이터는 `*.data.ts`로 분리(긴 콘텐츠 컴포넌트 분리 패턴).

### TakeHome / NonDev — 혜택의 구체화 (Before/After)
- **역할**: 졸업 시점에 손에 남는 것을 **대비로** 못 박기. 경쟁재(VOD 등)와 차별.
- **공식**: Before(흔한 방식의 한계) ↔ After(우리만의 결과). After는 `[~가 아니라] + [~를 남긴다]` 대구.
- granter 예시:
  - 헤딩: `보기만 하는 강의는 이제 그만, / 매일 쓰는 내 운영 대시보드를 남기세요`
  - Before: `영상 보고 따라 하기만 / 질문해도 답변은 며칠 뒤 / 예제 데이터로만 실습 / 수료증 한 장으로 끝`
  - After: `예제가 아니라 → 내 매출·재고로 직접 제작` / `수료증이 아니라 → 매일 쓰는 운영 자산`
- 팁: After의 `sub("~가 아니라") + main("~를 남긴다")` 2단 구조가 핵심. 반박 불가능한 대구를 만든다.

### Pricing — 얼리버드 긴급성
- **역할**: 가격 정당화 + "지금이 가장 싸다" + 선착순 희소성.
- **공식**:
  - 헤딩: `얼리버드 {N}명 한정, / 지금이 가장 저렴합니다`
  - 카드: `정가(취소선) → 얼리버드가` + `{N}% 할인 · {금액} ↓`
  - 보너스 레이어: 금액 없이 "추가로 받는 것"(granter: 그랜터 6개월 반값/무료).
  - 좌석 카운터: `{총}석 중 {잔여}석 남음` + 막바지 `곧 마감돼요 — {N}석만 남았습니다`.
- 팁: 희소성은 **진짜일 때만**. 카운터는 실제 결제수(`/api/counter`)에 연동 — 가짜 카운트다운은 신뢰 파괴 + 표시광고 이슈.
- 오픈 전엔 CTA를 `오픈까지 N일 HH:MM:SS` 실시간 타이머로(클릭 불가) → 오픈 후 `신청하기`로 전환.

### HowToJoin — 신청 절차
- **역할**: "복잡하지 않다"는 안심. 3~4단계 타임라인.
- **공식**: `1 가입/사전예약 → 2 안내 수신 → 3 결제/확정 → 4 시작`. 각 단계 1줄.

### FinalCTA — 클로징
- **역할**: 마지막 한 번 더. 망설임 직격 + 혜택 요약 + CTA.
- granter 예시: `아직 망설이고 계신가요? / 지금 사전예약하면 얼리버드 할인 우선 기회, 결제 시 무료 AI 스킬 자료까지 드립니다.`
- 템플릿: `{망설임 호명 질문} / 지금 {행동}하면 {혜택1}, {혜택2}까지 드립니다.`

### Faq — 반론 처리
- **역할**: 결제 직전 **남은 의심을 미리 해소**. 안 물어봐도 다들 궁금한 것.
- **반드시 포함할 반론 유형**:
  - 가격/환불: `환불 정책은? — 시작 7일 전까지 100% 환불, 이후 차등` (→ 환불정책 링크)
  - 자격/난이도: "비개발자도 되나요?" "어떤 환경이 필요한가요?"
  - 결과 의심: "정말 N주 후 {약속한 결과}가 되나요?" → 결과물=자산으로 재확인.
  - 대안 비교: "이미 {경쟁툴} 쓰는데?" → 차별점으로 전환.
  - 운영: "라이브 못 보면 녹화본?" "종료 후에도 쓸 수 있나?"
- **공식**: 질문은 **독자 입말 그대로**, 답변은 인정→해소 2단(`아닙니다/강제는 아닙니다. 다만…`). 방어적이지 않게.

## 2. 톤 & 보이스

| 항목 | 지침 |
|---|---|
| 인칭 | 독자 통증은 **1인칭 구어체**("~어요/~죠"), 약속·CTA는 **2인칭 명령형**("~하세요"). |
| 길이 | 헤딩 1~2줄, 카드 설명 1~2줄. 한 호흡에 읽히게. |
| 숫자 | 검증 가능한 수치만. 출처 암시 가능하면 함께. 없으면 쓰지 말 것. |
| 과장 | "최고/혁신/완벽" 류 형용사 금지. 결과물·기간·형태 같은 **사실**로 설득. |
| 이모지 | 본문 카피엔 거의 안 씀. 보너스/혜택 라벨에 1개 정도(🎁)까지만. |

## 3. AI slop 회피 (이거 쓰면 광고처럼 안 보이고 AI처럼 보인다)

생성형 카피에서 반복되는 냄새. 발견 즉시 교체:

- **"~의 세계로", "지금 바로 경험해보세요", "당신의 비즈니스를 한 단계 업그레이드"** 류 공허한 권유 → 구체 결과로.
- **삼단 형용사 나열**("빠르고, 강력하고, 직관적인") → 하나의 검증 가능한 사실로.
- **"혁신적인 솔루션", "원스톱", "올인원"** 같은 마케팅 클리셰 → 실제 동작 묘사로.
- **모든 문장이 같은 길이·같은 구조**(병렬 과잉) → 길이를 의도적으로 들쭉날쭉하게.
- **느낌표 남발** → 섹션당 최대 1개.
- **대상 불명확한 "여러분/고객님"** → 페르소나 라벨로 호명("월 300만 셀러").
- **번역투**("~을 가능하게 합니다", "~에 대해") → "~합니다", "~을".
- 테스트: 한 문장을 읽고 **"경쟁사 랜딩에 그대로 붙여도 말이 되면" 너무 일반적인 것** — 우리 상품에만 맞게 다시 쓴다.

## 4. 한국어 줄바꿈 톤 (keep-all + 의미 단위 `<br/>`)

한글은 단어 중간에서 줄이 깨지면 가독성이 급락한다. 두 가지를 함께 쓴다:

1. **CSS `word-break: keep-all`** — 어절(단어) 단위로만 줄바꿈. (디자인 스켈레톤에 이미 들어있음.)
2. **의미 단위 수동 `<br/>`** — 헤딩·중요 카피는 **읽는 호흡에 맞춰** 직접 끊는다.
   - granter 예시: `매출·재고·자금<br/>한 화면에서 관리하세요` (주어구 / 동사구로 분리)
   - 긴 한 줄이 어중간하게 접히는 것보다, 의미 단위로 끊어 **두 줄을 의도적으로** 만든다.
3. **줄바꿈 금지 구간은 `white-space: nowrap`** — 숫자+단위, 브랜드명 등 절대 안 깨질 덩어리.
   - granter 예시: `<span style={{ whiteSpace: "nowrap" }}>한 화면에서 관리하세요</span>`
4. 데이터 배열의 `desc`는 `\n`으로 끊고 CSS `white-space: pre-line`으로 렌더(WhyNow 패턴) — 카피와 줄바꿈을 데이터에 함께 보관.

⚠️ 모바일에서 수동 `<br/>`이 어색해질 수 있음 → 반응형으로 확인하고, 정 안 되면 `<br className="hidden md:inline" />` 식으로 분기.

## 5. 빠른 체크리스트 (카피 확정 전)

- [ ] Hero 한 줄로 "누구의 무슨 문제를 어떻게"가 전달되는가?
- [ ] 문제 섹션이 해결책보다 **먼저** 통증을 또렷이 하는가?
- [ ] 페르소나 라벨로 독자가 자기를 찾을 수 있는가?
- [ ] 혜택이 추상어가 아니라 **결과물/숫자/기간**으로 구체화됐는가?
- [ ] 긴급성(얼리버드/좌석)이 **진짜 데이터**에 연동되는가?
- [ ] FAQ가 결제 직전 주요 반론 5종(가격·자격·결과·대안·운영)을 덮는가?
- [ ] AI slop 표현이 남아있지 않은가? (§3 테스트)
- [ ] 헤딩 줄바꿈이 의미 단위로 끊겼고 모바일에서도 안 깨지는가?
