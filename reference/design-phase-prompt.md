# Phase 2 디자인 — 프롬프트 & 가이드 (비주얼 정체성 레이어)

> **역할 분담**: *구조*는 `design-system-skeleton.css`(박제된 코드)로 복제하고,
> *비주얼 정체성*(색·무드·타이포)만 이 프롬프트로 매번 새로 정한다.
> 스켈레톤은 토큰 "값"만 비어 있으니, 이 단계의 결과 = 그 값들 + 섹션별 비주얼 방향.

## 산출물
1. **확정 팔레트**(--ink/--paper/--signal/--signal-text/--rule/--mute 값)
2. **폰트 페어링**(display/body/mono + 로딩 방식)
3. **무드 한 줄**(예: "신뢰형·차분·다크 강조" vs "역동·하이콘트라스트")
4. → `design-system-skeleton.css`의 `:root`에 그대로 주입.

---

## STEP 1 — 레퍼런스 수집 (기획 P0와 병행 권장)
사용자에게 받거나 직접 수집:
- 유사·경쟁 강의 랜딩 3~5개(패스트캠퍼스·인프런·클래스101 등) 스크린샷/URL.
- 디자인 인스피레이션(Dribbble·Behance·awwwards) 2~3개.
- 브랜드 기존 자산(로고·기존 사이트·가이드)이 있으면 색·폰트 추출.

> 받을 게 없으면 사용자에게 **3가지만** 물어라(AskUserQuestion):
> ① 무드(신뢰/전문 vs 역동/트렌디) ② 메인 강조색 방향(웜/쿨/뉴트럴) ③ 다크 비중(라이트 위주 vs 다크 강조).

## STEP 2 — 톤앤매너 결정
레퍼런스에서 추출 → 결정:
- **강조색(--signal)**: 1개. CTA·포인트 전용. 너무 흔한 채도/명도 피하고 대비 확보.
- **--signal-text**: 강조색을 텍스트로 쓸 때용 더 진한 톤(밝은 배경 위 4.5:1 대비).
- **--ink/--paper**: 순흑/순백 대신 살짝 톤다운(예 #1A1A1A / #FAFAF7)이 고급스럽다.
- **--rule/--mute**: ink 계열에서 파생.
- **폰트**: 한국어면 Pretendard 기본 안전. 개성 필요 시 display만 교체(본문은 가독성 우선).

## STEP 3 — 디자인 스킬 연계 (선택, 품질↑)
- `ui-ux-pro-max` — 스타일/팔레트/폰트페어링 후보 제안받기(레퍼런스 무드 입력).
- `design-shotgun` 또는 `design-html` — 같은 섹션을 3~5개 시안으로 다중 생성 → 비교.
- `design-review` / `design-consultation` — 생성 시안 비평·선택.
- 결과의 색·폰트를 STEP 2 토큰으로 환원.

## STEP 4 — 토큰 주입 & 검증
- 결정값을 `design-system-skeleton.css` `:root`에 기입(변수명·구조는 절대 변경 금지).
- 검증: ① 강조색 대비(WCAG AA 4.5:1) ② 다크섹션에서 --signal 가독성 ③ 버튼 hover/disabled 상태 ④ 라이트/다크 섹션 전환 자연스러움.

## 📋 마스터 프롬프트 템플릿 (복붙해서 사용)

```
강의 랜딩 디자인 시스템의 "비주얼 정체성"을 정한다.
구조는 design-system-skeleton.css(검증된 스캐폴드)를 쓰므로, 너는 토큰 "값"과 무드만 결정한다.

[입력]
- 강의/브랜드: <이름·주제>
- 타깃: <대상>
- 무드: <신뢰/전문 | 역동/트렌디 | 미니멀 | ...>
- 레퍼런스: <URL/스샷 또는 "없음 → 무드 기반 제안">
- 다크 비중: <라이트 위주 | 다크 강조 혼합>
- 기존 브랜드색: <있으면 HEX, 없으면 "자유">

[해줄 것]
1. 무드를 한 줄로 정의.
2. 팔레트 확정 (HEX):
   --ink / --paper / --signal / --signal-text / --rule / --mute
   각 색의 선택 이유 1줄 + 강조색 WCAG AA 대비 확인.
3. 폰트 페어링: display / body / mono + 로딩 방식(웹폰트 CDN or next/font).
4. 위 값을 design-system-skeleton.css :root 에 주입한 최종 CSS 블록 출력.
5. (선택) 라이트/다크 섹션 각 1개 미니 목업으로 색 적용 미리보기.

[금지]
- skeleton의 변수명·공통클래스·reveal·반응형·접근성 규칙 변경 금지(구조는 고정).
- 강조색을 본문 대량 텍스트에 쓰지 말 것(포인트 전용).
```

## 결정 트리 — 코드참고 vs 프롬프트 (요약)
```
구조(토큰구조·폰트로딩·.section/.reveal·반응형·접근성)  → design-system-skeleton.css 복제
비주얼(색·무드·타이포)                                  → 이 프롬프트 + 디자인 스킬
완성 섹션(맥북 스크럽·인물 스포트라이트)                  → assets/ 드롭인
```
라이브 granter 레포는 참조하지 않는다(움직이는 표적). 스켈레톤이 단일 출처.
